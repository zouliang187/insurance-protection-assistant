# 网站信息架构

## 1. 首页

首页的目标不是营销，而是让用户迅速判断：

- 这个平台是否可信
- 我能在这里解决什么问题
- 我应该从哪里开始

建议首屏结构：

- 平台一句话定位
- 三个入口：
  - 给父母买保险
  - 给自己和家庭做保障体检
  - 学保险基础知识
- 公益与非销售声明
- 数据来源和更新机制入口

## 2. 核心导航

### 保障体检

功能：

- 家庭成员录入
- 医保状态确认
- 健康状态初筛
- 预算区间
- 风险责任识别
- 输出保障优先级

页面：

- `/checkup`
- `/checkup/result`

### 给父母买保险

功能：

- 父母年龄选择
- 健康状况选择
- 是否有医保
- 是否有慢病或既往症
- 输出可考虑路径

页面：

- `/parents`
- `/parents/paths`

### 保险入门

内容：

- 医保到底能报什么
- 百万医疗险是什么
- 惠民保适合谁
- 意外险怎么买
- 重疾险是不是必须买
- 定期寿险适合谁
- 健康告知怎么读
- 理赔为什么会被拒

页面：

- `/learn`
- `/learn/[slug]`

### 产品库

功能：

- 产品筛选
- 条款字段对比
- 保障责任说明
- 适合/不适合人群
- 数据来源和更新日期

页面：

- `/products`
- `/products/[id]`
- `/compare`

### AI 条款助手

功能：

- 上传或粘贴保险条款
- 自动提取责任、免责、续保、等待期、健康告知
- 生成普通话解释
- 生成投保前核对清单

页面：

- `/clause-assistant`

### 公益说明

内容：

- 为什么做这个项目
- 不接受哪些商业合作
- 数据如何维护
- 错误如何反馈
- 合规声明

页面：

- `/about`
- `/methodology`
- `/feedback`

## 3. 用户路径

### 路径 A：年轻人为父母配置

1. 首页点击“给父母买保险”
2. 输入父母年龄、医保、健康情况
3. 得到路径判断
4. 阅读对应险种解释
5. 查看产品字段对比
6. 生成投保前问题清单

### 路径 B：普通家庭保障体检

1. 首页点击“家庭保障体检”
2. 添加家庭成员
3. 填写预算和责任
4. 得到家庭保障优先级
5. 查看每个成员的建议险种
6. 跳转到科普或产品对比

### 路径 C：从文章进入

1. 用户通过转发文章进入
2. 文章解释一个具体问题
3. 文末引导做保障体检或父母保险选择
4. 输出下一步行动清单

## 4. 首版 MVP 页面

第一版建议只做这些页面：

- 首页
- 保险入门列表
- 3-5 篇核心文章
- 给父母买保险路径页
- 家庭保障体检问卷
- 保障体检结果页
- 产品字段对比表
- 关于与公益声明

## 5. 页面文案基调

应该使用：

- 朴素
- 克制
- 解释清楚
- 不制造焦虑
- 不承诺结果

避免使用：

- “必买”
- “闭眼入”
- “全网第一”
- “不买就亏”
- “保证赔”
- “最后机会”
