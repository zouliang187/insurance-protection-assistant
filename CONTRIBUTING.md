# Contributing

感谢你愿意改进保障先知。这个项目关注保险科普、资料核验和保障路径理解，贡献时请优先保证准确、克制和可复核。

## 适合贡献的内容

- 补充国家医保局、金融监管部门、保险公司条款等权威资料。
- 改进父母保险选择器的问题设计、结果解释和核对清单。
- 建立产品数据字段样例，包括投保年龄、健康告知、续保条件、免赔额、既往症责任等。
- 修复页面可访问性、移动端布局和交互问题。
- 补充合规边界、免责声明和资料更新时间。

## 本地查看

直接打开 `prototype/index.html`，或在仓库根目录启动任意静态文件服务后访问 `prototype/`。

## 资料来源要求

提交资料或结论时，请尽量包含：

- 来源链接。
- 来源类型，例如监管文件、官方条款、统计公报、研究报告。
- 发布时间或资料更新时间。
- 数据口径。
- 是否已经人工复核。
- 可能偏差或适用范围。

## Pull Request 建议

- 一次 PR 只解决一个明确问题。
- 涉及保险条款、监管规则、统计数据时，请附上可复核来源。
- 涉及产品展示或路径推荐时，请明确说明合规边界和用户误解风险。
- 不提交销售链接、佣金导向榜单、夸大收益或承诺理赔的内容。

## 维护原则

- 公益优先，不以销售转化为目标。
- 保障优先，先解释风险缺口，再讨论产品类型。
- 透明优先，公开资料来源、规则和限制。
- 合规优先，不替代持牌专业人员出具个性化保险建议。
